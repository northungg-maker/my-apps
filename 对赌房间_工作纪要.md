## 七、规则条款 ✅ 已定稿

### 📜 对赌房间契约规则

**一、契约性质**
1. 本契约为参与各方自愿签署的电子协议。
2. 系统仅提供契约管理工具，不承担裁决责任。
3. 签署后各方需按约定履行义务。

**二、监督方**
1. 监督方由各方共同选择，请选择您信任的人。
2. 监督方负责跟踪执行情况，协助调解争议。
3. 监督方的建议裁决仅供参考，不具有强制执行力。
4. 如对监督方裁决有异议，建议优先协商解决。

**三、确认密码**
1. 各方签署时需设置 4 位确认密码
2. 契约到期后，各方需输入自己的密码确认最终结果
3. 请妥善保管密码，遗忘将无法参与确认

**四、结果确认**
1. 契约到期后进入「待确认」状态
2. 各方需各自输入密码确认结果
3. 全部确认后契约状态确定

**五、争议处理**
1. 若确认结果不一致，契约进入「争议」状态
2. 监督方可给出建议裁决，协助调解
3. 争议优先协商解决，协商不成可走法律途径
4. 系统不承担争议裁决责任

**六、违约责任**
按契约约定的「代价」条款执行，各方签字即表示接受。

### 2026-04-06 Firebase 权限问题详解

**现象：**
- 签署流程正常走通（规则弹窗 → 密码弹窗 → 点击确认）
- 但最后一步卡住，房间未创建
- 控制台报错：`FirebaseError: [code=permission-denied]: Permission denied`

**原因：**
- Firebase Realtime Database 默认规则：`".write": "auth != null"`
- 要求用户必须登录认证才能写入数据
- 当前应用未接入 Firebase Auth，所以写入被拒绝

**解决：**
1. 通过 Browser 打开 Firebase Console
2. 进入项目 `bet-room-722bd` → Realtime Database → 规则
3. 将规则改为：
```json
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```
4. 发布规则（会提示安全警告，确认发布）

**测试结果：**
- ✅ 单人模式完整流程通过
- ✅ 房间号生成正常（如 `R34QJH`）
- ✅ 密码保存到 Firebase 正常
- ✅ 状态 `locked` 正常显示

### 2026-04-06 流程重构（第十二节）

**发现问题根因：**
- 原 `initiateSign('partyA')` 调用 `doSignContract`，需要已有 `currentRoomId`
- 但创建新契约时，`currentRoomId` 只有 `finalizeContract` 才生成
- 形成死循环：没房间号无法签署，不签署无法生成房间号

**解决方案：**
1. 新建 `initiateCreateAndSign()` 函数处理"创建契约"流程
2. 新建 `doCreateAndLock(password)` 函数整合规则+密码+创建房间
3. 区分两种场景：
   - **创建新契约：** `initiateCreateAndSign()` → 规则 → 密码 → 直接生成房间
   - **加入已有契约：** `initiateSign(role)` → 规则 → 密码 → 签署（已有房间号）

**代码修改：**
- 签署按钮：`onclick="initiateSign('partyA')"` → `onclick="initiateCreateAndSign()"`
- 新增函数：`initiateCreateAndSign()`、`doCreateAndLock(password)`

**单人模式流程：**
```
点击"确认签署" → initiateCreateAndSign()
    → 检查签名
    → 显示规则弹窗
    → 用户同意 → 显示密码弹窗
    → 用户输入密码 → doCreateAndLock(password)
    → 生成房间号 + 保存密码 + 状态改为 locked
    → 跳转到房间页面
```

**双方模式流程：**
```
点击"确认签署" → initiateCreateAndSign()
    → 检查签名
    → 显示规则弹窗
    → 用户同意 → 显示密码弹窗
    → 用户输入密码 → doCreateAndLock(password)
    → 生成房间号 + 保存密码 + 甲方签署状态 = true
    → 状态 = pending（等待乙方）
    → 跳转到房间页面
```

### 签署流程设计
```
用户点击签署按钮
    ↓
initiateSign(role) 被调用
    ↓
showRulesModal() 显示规则弹窗
    ↓
用户勾选同意 → confirmRules() 关闭弹窗，调用 onConfirm
    ↓
showPasswordModal(role) 显示密码弹窗
    ↓
用户输入密码并确认 → submitPassword() 验证后调用 onConfirm(password)
    ↓
doSignContract(role, password) 执行实际签署
    ↓
Firebase 更新：保存密码、签署状态、时间戳
```
