# 小龙虾 V4.0 修改执行方案（完整打包版）

## 核心目标

1. 所有结论必须有数据支撑，不允许"均衡发展""无明显"等模糊表述。
2. 输出面向普通用户：第一层 10 秒即可形成一句话判断。
3. 支持按需展开，二三层用于追溯原因和提供完整数据（JSON）。
4. 三个模块独立：执行、建议、风险，互不干扰。

---

## 修改方案清单

### 1️⃣ 优势/短板阈值调整（最简单，5分钟）

- 当前规则：阈值 = 平均值 ±1
- 新规则：

```javascript
// 输出 Top1/Bottom1，阈值 ±0.3
function getStrengthsWeaknesses(scores, overallAvg) {
  let threshold = 0.3;
  let strengths = [];
  let weaknesses = [];

  scores.forEach(s => {
    if(s.value >= overallAvg + threshold) strengths.push(s);
    if(s.value <= overallAvg - threshold) weaknesses.push(s);
  });

  // 如果没有明显差异，强制输出最高和最低
  if(strengths.length === 0) strengths.push(scores.reduce((a,b)=>a.value>b.value?a:b));
  if(weaknesses.length === 0) weaknesses.push(scores.reduce((a,b)=>a.value<b.value?a:b));

  return { strengths, weaknesses };
}
```

### 2️⃣ 核心判断模块（10分钟）

- 功能：聚合已有结论，生成一句话核心判断。
- 示例函数：

```javascript
function getCoreJudgment(scores) {
  // scores: 子系统评分列表
  // 规则：综合分析最高认知+执行能力+风险，输出一句话
  let cognition = scores.find(s=>s.name==="认知系统").value;
  let execution = scores.find(s=>s.name==="执行能力").value;
  let risk = scores.find(s=>s.name==="风险等级").value;

  let judgment = `该用户综合表现${risk>=4?"优秀":"中等"}，认知${cognition>=4?"强":"一般"}，执行${execution>=4?"高效":"中等"}`;
  return judgment;
}
```

### 3️⃣ 决策引擎引用数据（10分钟）

- 每条结论都必须显示"支撑数据 + 限制条件"

```javascript
function generateDecisionEngine(scores) {
  return {
    可合作: {
      结论: scores.find(s=>s.name==="核心层").value >=3.5 ? "适合" : "视情况",

      限制: scores.find(s=>s.name==="学习能力").value < 3.0 ? "学习能力较低需监督" : ""
    },
    可深交: {...},
    可托付: {...}
  }
}
```

### 6️⃣ 核心约束

1. 所有输出必须有数据来源。
2. 第一层 10 秒可形成一句话判断。
3. 禁止模糊表述、重复输出。
4. 建议、风险、执行完全独立。

---

## 7️⃣ 已识别问题与原因

| 问题 | 解释 |
|------|------|
| 示例代码报错 | 命名、返回结构或 DOM 操作与现有环境不一致 |
| 输出信息重复 | 第二层解释应追溯原因，而不是重新输出第一层结论 |
| 决策引用数据不统一 | 结论需要明确"支撑 + 限制"，当前格式不一致 |
| 风险列表混乱 | 用户看不到统一风险等级，需先结论后追溯来源 |

---

## 8️⃣ 标准执行原则（达成一致）

1. **不改变原有逻辑**
   - 保留现有函数、数据结构，只修改输出和阈值逻辑。
2. **分模块执行**
   - 执行、建议、风险模块独立，不混合。
3. **结论优先，层层追溯**
   - 第一层：一句话定性判断（普通用户）
   - 第二层：详细解释/原因（追溯第一层结论）
   - 第三层：完整数据（JSON，可导出给 AI/专业人士）
4. **引用数据**
   - 每条结论必须明确数据来源，不允许模糊表达。
5. **阈值调整**
   - 优势/短板阈值由 ±1 改为 ±0.3 或强制输出 Top1/Bottom1，确保一定会输出差异。
