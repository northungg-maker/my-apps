   - 2.0-2.5 → 限制性短板
   - 2.5-3.0 → 功能性短板

2. **getSystemCategory(sys)** - 系统类别判断
   - 用于补偿判定

3. **canStrengthCoverWeakness()** - 优势覆盖短板判定
   - 同层同类可补偿
   - 战略层可支撑核心层

4. **getRiskLevel()** - 统一风险等级
   - 0分存在 → 极高风险
   - 最低分≤2.0 → 高风险
   - 最低分2.0-3.0 → 中风险
   - 最低分≥3.0 → 低风险

## 崩溃原因分析（重要经验）

执行了三次崩溃，根本原因：

1. **函数过大** - `updateDecisionEngine()` 200+行，5个判断，嵌套太深
2. **变量作用域复杂** - 多处依赖，返回结构不确定
3. **独立计算重复** - 风险等级在两处分别计算
4. **一次性改太多** - 8个部分同时改，无法定位错误

**解决方案**: 分步执行 + 每步验证 + 模块化新增函数
