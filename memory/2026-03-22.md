## 技术笔记

### iOS Safari 复制兼容方案
```javascript
function copyToClipboard(text) {
    const textarea = document.createElement('textarea');
    textarea.value = text;
    textarea.style.position = 'fixed';
    textarea.style.top = '0';
    textarea.style.left = '0';
    textarea.style.width = '100%';
    textarea.style.height = '200px';
    textarea.style.opacity = '1';
    textarea.style.zIndex = '9999';
    textarea.style.fontSize = '16px';
    document.body.appendChild(textarea);
    textarea.select();
    document.execCommand('copy');
    document.body.removeChild(textarea);
}
```

### 通用表单导出方法

### 方向分类边界问题（2026-03-22 23:13 讨论）

**问题本质**：
一篇脚本可能涉及多个方向，归类时边界模糊。

**举例**：
- "为什么越努力越穷" → 心理学？经济学？行为经济学？
- "社交焦虑的经济学解释" → 心理学？经济学？

**可能的解决方案**：

| 方案 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| 多标签 | 一条内容可选多个方向 | 灵活 | 统计时会重复计算 |
| 主次标签 | 主方向+次方向 | 明确主攻方向 | 操作复杂 |
| 内容时确定边界 | 写脚本时就明确单一方向 | 简单 | 可能限制内容创作 |
| 细分方向 | 把方向拆得更细（如：行为经济学单独列出） | 减少模糊 | 方向太多 |

**待用户确认处理方式**


### 方向标签最终决定（2026-03-22 23:21）

**决定**：保持单选，先跑起来

**理由**：
1. 统计清晰，不会虚高
2. 操作简单
3. 方向验证的本质是判断"这个方向值不值得继续"
4. 真正跨方向的内容其实不多

**方向确定规则**：
- 脚本生成时就确定单一方向
- 方向 = 内容的核心切入角度
- 如果内容跨领域，选最核心的那个

**标签管理**：
- 工具支持自定义输入
- 定期（每月）盘点标签使用情况
- 使用率低的标签合并或清理
- 保持核心标签在 7-10 个

### 方向验证执行规则补充（2026-03-22 23:41）

**核心原则**：在一个方向验证清楚之前，不许换方向。

**判断标准**：
| 测试数 | 有效率 | 结论 |
|--------|--------|------|
| <5条 | - | 数据不足，继续发 |
| ≥5条 | ≥40% | 方向可行 |
| ≥5条 | <20% | 有问题，调整表达 |
| ≥10条 | 0有效 | 建议放弃 |

**调整vs换方向**：
- 调整 = 同一大类切入点不同 → 继续当前方向
- 换方向 = 完全不同大类 → 需先验证完当前

**标签管理**：
- 一个方向验证完再做下一个
- 同时只测1-2个方向
- 标签数量由执行纪律控制
