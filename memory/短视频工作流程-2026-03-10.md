# 短视频内容生成工作流程

## 零、输出位置

**⚠️ 默认存放目录（强制）：** `/Users/tolo/Documents/openclaw/output`

**禁用其他位置** — 不要存到 workspace/短视频脚本 或其他地方。

每篇脚本立即保存到独立文件，避免对话上下文堆积。

### 文件命名格式
`短视频脚本_序号_标题关键词.md`
例：`短视频脚本_021_病理性独立.md`

### 必须包含
- **原文链接** — 每篇必须贴来源URL，方便验证是否真实爆款
- **不额外输出** — 只输出脚本本身，不主动加思维导图等，除非明确要求

---

## 一、稳定运行原则

### 1. 分批生成
- ❌ 不要一次生成多篇，容易压缩上下文、质量下降
- ✅ 每次生成 1 篇 → 写文件 → 清上下文 → 下一篇
- 大任务（如10篇）依次独立完成，不可重复

### 2. 确保真实爆款
- 搜索必须返回原文链接，否则 AI 可能"假爆款"
- 标准：阅读 >10万，点赞 >1万，或大量转载

### 3. 控制上下文
- 16k tokens 容量，一次生成太多会被截断
- 每篇文章独立处理 → 输出独立文档
- **监控 tokens：** 超过 90% 时停止，先保存或开新会话

### 4. 及时拆分会话
- 大任务分多个独立会话或子任务
- 旧内容不会影响新任务的上下文

### 5. 明确脚本结构
```
钩子 → 案例 → 心理学解析 → 反转结论 → 互动问题
```
- 字数：800-1200 字
- 平台：抖音、小红书、B站口播

---

## 二、Prompt 模板

**任务：**
搜索心理学爆款内容，并改写为适合中国短视频平台的内容。

**要求：**
1. 来源真实文章：
   - 知乎、公众号、Medium、Reddit、PsychologyToday
2. 必须是爆款：
   - 阅读 >10万 或 点赞 >1万
   - 或大量转载
3. 方向：
   - 情感关系、人际关系、自我成长

**每篇文章执行：**
- 步骤1：给出原文标题 + 来源链接
- 步骤2：总结爆款结构
- 步骤3：彻底改写 → 抖音/小红书短视频脚本

**格式：**
- 开头钩子
- 故事案例
- 心理学解释
- 反转结论
- 互动问题

**字数：** 800-1200字
**输出：** 每篇独立文档，每次调用生成 2~3 篇 → 循环 5 次

---

## 三、流水线化（每步独立执行）

每次只做一步，减少上下文压力：

| 步骤 | 任务 | 输出 |
|------|------|------|
| 1 | 爆款文章搜索 | 标题 + 链接（独立会话） |
| 2 | 分析结构 | 拆解结果写文件 |
| 3 | 改写脚本 | 独立文档存档 |
| 4 | 生成标题/封面 | 单独输出 |

**关键：** 每步完成后保存文件，上下文干净再进下一步。

## 四、配音生成流程

### 流程概览
```
文案优化 → 给我 → 我出配音文件 → 你剪辑成片
```

### 我能做的
- **配音生成**：文案给我，用 TTS 生成音频文件
- **配音风格选择**：可试听不同声音，选一个固定的"品牌声音"
- **输出格式**：MP3 / WAV

### 需要你完成的
- **视频画面**：拍摄或找素材
- **剪辑合成**：配音 + 画面组合
- **字幕/特效**：后期加工

### 试点建议
先选一篇文案 → 优化好 → 试配音 → 定风格 → 批量处理

## 五、Subagent 使用规范

### 什么时候用 Subagent
- **批量生成：** 一次生成多篇脚本
- **长任务：** 可能跑几十分钟的任务
- **隔离需求：** 需要独立上下文，不污染主 session

### 用法
- 指令：「用 subagent 帮我处理 XXX」
- 效果：派子代理独立跑，主 session 保持轻量
- 完成后自动汇报结果

### 对比

| 方式 | 上下文 | token 消耗 | 适用场景 |
|------|--------|-----------|----------|
| 主 session | 累积 | 高 | 简单对话、快速问答 |
| Subagent | 隔离 | 低 | 批量处理、长任务 |
