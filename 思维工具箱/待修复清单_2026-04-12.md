## 一、发现的问题

### 问题1：功能介绍不够丰富

**现象：**
- 每个工具的功能说明只有一行 `tool-why`
- 使用指南藏在折叠里，用户可能不会点开

**影响：**
- 用户不知道工具能解决什么问题
- 降低了工具的使用意愿

---

### 问题2：结构变化后阶段逻辑弱化

**现象：**
- 顶部导航是平铺12个按钮
- 进入工具后看不出属于哪个阶段

**影响：**
- 用户不知道自己在"想清楚"、"想明白"还是"做决策"阶段
- 失去了原方案的阶段引导逻辑

---

### 问题3：诊断后缺少行动指引

**影响：**
- 用户知道问题在哪，但不知道怎么办
- 工具停留在"诊断"，没有"治疗"

### 修改项3：补充诊断后的行动指引

**价值判断三段论：**
```
if 分歧在事实层 → 建议：收集数据、查找证据
if 分歧在标准层 → 建议：寻找第三方标准、建立共识
if 分歧在价值层 → 建议：承认无法调和，搁置争论或分道扬镳
```

**MECE原则：**
```
if 有遗漏 → 建议：补充遗漏的分支
if 有重叠 → 建议：重新划分边界，合并重复项
if 完整无重叠 → 建议：结构清晰，可以继续推进
```

**结构化归因：**
```
根据各因素贡献度排序
→ 建议：优先解决贡献度最高的因素
```

**权衡论证表：**
```
计算各选项总权重分
→ 建议：选择总分最高的选项
```

**信息核真：**
```
if 可靠性高 → 建议：可以采纳
if 可靠性存疑 → 建议：需要更多信源交叉验证
if 可靠性低 → 建议：谨慎使用或不予采纳
```

**文件：** `/Users/tolo/Library/Mobile Documents/com~apple~CloudDocs/openclaw/思维工具箱/index.html`
