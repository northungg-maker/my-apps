# 思维工具箱 - 项目档案

## 项目概述

**来源：** 学会思考2.pdf（批判性思维工具包）
**目标：** 将PDF中的12个思维工具转化为交互式网页程序
**用途：** 个人思考辅助 + 自媒体内容生成参考

---

## 独立思考全流程

```
输入端 → 想清楚 → 想明白 → 做决策 → 输出端
```

### 阶段一：想清楚
| 类型 | 工具 | 为什么要用它 |
|------|------|-------------|
| 问题分析 | 四步法 | 面对复杂问题时，帮你拆解成可操作的思考路径 |
| 信息核真 | 信息核真流程 | 面对纷繁信息时，帮你辨别真伪、追溯源头 |
| 论证评估 | ARG三标准 | 评价一个论证是否有说服力 |
| 论证评估 | 权衡论证表 | 面对两难选择时，帮你看清利弊、找到兼顾方案 |

### 阶段二：想明白
| 类型 | 工具 | 为什么要用它 |
|------|------|-------------|
| 结构化表达 | 金字塔模型 | 想把想法说清楚时，帮你结构化输出 |
| 结构化表达 | MECE原则 | 分类时帮你检查"不重不漏"，避免混乱 |
| 因果分析 | 结构化归因 | 想弄清一件事为什么发生时，系统梳理原因 |
| 因果分析 | 实验法/因果分析表 | 想判断因果关系时，用对照实验逻辑验证 |

### 阶段三：做决策
| 类型 | 工具 | 为什么要用它 |
|------|------|-------------|
| 方案评估 | 目标-手段评估表 | 评估一个方案是否能达成目标 |
| 方案评估 | 赋值法 | 多个方案纠结时，用量化帮你决策 |
| 价值判断 | 价值判断三段论 | 争论不下时，帮你看清分歧在哪个层面 |

### 参考卡
| 工具 | 为什么要用它 |
|------|-------------|
| 常见逻辑谬误 | 识别日常对话中的逻辑陷阱，避免被带节奏 |

## 技术方案

**前端：** 纯HTML + CSS + JavaScript（无后端依赖）
**部署：** GitHub Pages / 本地直接打开
**特点：**
- 单页面应用（SPA），工具切换无刷新
- 数据本地存储（localStorage）
- 可导出分析结果为Markdown/文本

## Token 规划

每个工具开发约 2000-4000 tokens（代码+调试）
总计：12个工具 × 3000 = **约36,000 tokens**

**策略：**
- 主session只做框架和简单工具
- 复杂工具用 subagent 开发
- 每完成一个工具，更新状态文件
